# Security Policy

This repository is intended for documentation, lab use, and research-oriented deployments.

## Reporting a vulnerability

If you discover a security issue, please report it privately before opening a public issue.

## Basic recommendations

- Do not expose SIP services directly to the public internet without proper controls
- Replace example passwords before any real deployment
- Restrict SSH access
- Use VPN for remote administration when possible
- Review firewall rules for SIP and RTP traffic
- Avoid using default credentials in production environments

## Scope

The configurations in this repository are examples for educational and lab purposes.
They are not production-hardened by default.
